🎯 O Sistema de Detecção de Fraudes

Beatriz era cientista de dados em um banco digital. Seu modelo de detecção de fraudes estava com 98% de acurácia — e o diretor estava comemorando. Mas Beatriz sabia que havia algo errado: das 10.000 transações diárias, apenas 50 eram fraudes (0,5%). Um modelo que simplesmente classificasse tudo como legítimo já teria 99,5% de acurácia.

O problema real era probabilístico: dado que uma transação tem certas características suspeitas, qual a probabilidade de ser fraude? Para responder isso corretamente, Beatriz precisava dominar probabilidade condicional, o Teorema de Bayes e as distribuições que descrevem eventos raros.

Fundamentos de Probabilidade

Probabilidade Clássica

A probabilidade de um evento \(A\) é a razão entre os casos favoráveis e o total de casos igualmente prováveis:

$$P(A) = \frac{\text{casos favoráveis}}{\text{total de casos}}$$

Probabilidade Condicional

A probabilidade de \(A\) ocorrer dado que \(B\) já ocorreu. Essa é a base do raciocínio bayesiano:

$$P(A \mid B) = \frac{P(A \cap B)}{P(B)}$$
💡 No contexto de Beatriz \(P(\text{Fraude} \mid \text{Transação Suspeita})\) é muito diferente de \(P(\text{Fraude})\). A probabilidade condicional atualiza nossa crença com base em evidências — exatamente o que um modelo de ML faz.

Teorema de Bayes

Permite inverter a probabilidade condicional — calcular \(P(\text{causa} \mid \text{efeito})\) a partir de \(P(\text{efeito} \mid \text{causa})\):

$$P(A \mid B) = \frac{P(B \mid A) \cdot P(A)}{P(B)}$$

🔬 Bayes na Prática: O Paradoxo da Acurácia

Suponha: P(Fraude) = 0,5%, P(Suspeita | Fraude) = 90%, P(Suspeita | Legítima) = 5%.

Calculando P(Fraude | Suspeita) pelo Teorema de Bayes:

$$P(\text{Fraude} \mid \text{Suspeita}) = \frac{0{,}90 \times 0{,}005}{0{,}90 \times 0{,}005 + 0{,}05 \times 0{,}995} \approx 8{,}3\%$$

Mesmo com um alerta suspeito, a probabilidade de fraude é apenas 8,3%! Isso explica por que sistemas de fraude geram muitos falsos positivos — e por que métricas como Precision e Recall são mais relevantes que acurácia nesse contexto.

Distribuições Discretas

Distribuição de Bernoulli

O caso mais simples: um único experimento com dois resultados possíveis (sucesso/fracasso). É o átomo de todas as distribuições discretas.

$$P(X = k) = p^k (1-p)^{1-k}, \quad k \in \{0, 1\}$$

Exemplo: Uma transação é fraude (1) ou não (0), com \(p = 0{,}005\).

\(E[X] = p\)    \(Var(X) = p(1-p)\)

Distribuição Binomial

Generaliza Bernoulli para \(n\) experimentos independentes. Conta o número de sucessos em \(n\) tentativas.

$$P(X = k) = \binom{n}{k} p^k (1-p)^{n-k}$$

Exemplo: Em 1.000 transações, quantas serão fraude? \(X \sim B(1000, 0{,}005)\).

\(E[X] = np\)    \(Var(X) = np(1-p)\)

Distribuição de Poisson

Modela o número de eventos raros em um intervalo fixo de tempo ou espaço. Ideal quando \(n\) é grande e \(p\) é pequeno.

$$P(X = k) = \frac{e^{-\lambda} \lambda^k}{k!}, \quad \lambda = np$$

Exemplo: Número de fraudes por hora em um sistema com taxa média \(\lambda = 2\) fraudes/hora.

Comparação entre Binomial(n=20, p=0.1) e Poisson(λ=2). Para n grande e p pequeno, as distribuições convergem.

Distribuições Contínuas

Distribuição Normal (Gaussiana)

A mais importante da estatística. Simétrica, definida por média \(\mu\) e desvio padrão \(\sigma\).

$$f(x) = \frac{1}{\sigma\sqrt{2\pi}} e^{-\frac{1}{2}\left(\frac{x-\mu}{\sigma}\right)^2}$$

A Regra Empírica (68-95-99.7): 68% dos dados estão em \(\mu \pm \sigma\), 95% em \(\mu \pm 2\sigma\), 99.7% em \(\mu \pm 3\sigma\).

Distribuição Exponencial

Modela o tempo entre eventos em um processo de Poisson. Tem a propriedade de ausência de memória.

$$f(x) = \lambda e^{-\lambda x}, \quad x \geq 0$$

Exemplo: Tempo entre duas fraudes consecutivas, dado que ocorrem a uma taxa \(\lambda\).

Distribuição Uniforme

Todos os valores no intervalo \([a, b]\) têm a mesma probabilidade. Base para geração de números aleatórios e inicialização de pesos em redes neurais.

$$f(x) = \frac{1}{b-a}, \quad a \leq x \leq b$$

Normal(μ=0, σ=1), Exponencial(λ=1) e Uniforme(0,1) — três formas completamente diferentes de distribuir probabilidade.

🔬 Distribuições em Algoritmos de ML

A escolha da distribuição não é apenas teórica — ela define a função de perda do seu modelo:

  • Normal → Regressão Linear: Assume que os erros seguem \(\mathcal{N}(0, \sigma^2)\), o que leva ao método dos mínimos quadrados.
  • Bernoulli/Binomial → Regressão Logística: Modela a probabilidade de classes binárias via função sigmoide.
  • Poisson → Regressão de Poisson: Para modelar contagens (número de cliques, vendas por dia, fraudes por hora).
  • Exponencial → Modelos de Sobrevivência: Para modelar tempo até um evento (churn, falha de equipamento).

Quando você escolhe uma função de perda, está implicitamente assumindo uma distribuição para os seus dados. Entender isso é o que separa quem usa ML de quem entende ML.

Teorema Central do Limite

Um dos resultados mais poderosos da estatística: independentemente da distribuição original dos dados, a média amostral de amostras suficientemente grandes converge para uma distribuição Normal.

$$\bar{X} \xrightarrow{d} \mathcal{N}\left(\mu, \frac{\sigma^2}{n}\right) \quad \text{quando } n \to \infty$$
💡 Por que isso importa para ML? O TCL justifica o uso de testes de hipótese e intervalos de confiança mesmo quando os dados originais não são normais. É também a base teórica para técnicas como bootstrap e validação cruzada — você está estimando a distribuição da performance do modelo via reamostragem.

📝 Resumo para Levar na Mochila

  • Probabilidade condicional é a base do raciocínio com evidências — e de todo modelo supervisionado.
  • Teorema de Bayes inverte a condicional: de P(efeito|causa) para P(causa|efeito).
  • Bernoulli/Binomial para eventos binários; Poisson para eventos raros em intervalos.
  • Normal para erros e médias; Exponencial para tempos entre eventos.
  • A distribuição que você assume define a função de perda do seu modelo.
  • TCL garante que médias amostrais são normais — base dos testes estatísticos.
